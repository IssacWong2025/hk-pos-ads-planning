三、Meta 廣告形式管理層速覽

第一層：一句話畫面（看到什麼，點了去哪）

  • Feed：像一般滑動貼文中的贊助內容，點擊後通常進入表單或訊息頁。
  • Stories：全螢幕直式畫面，點擊後直接跳轉到表單或對話入口。
  • Reels：短影片流中的插入廣告，點擊後多數導向訊息或落地頁。
  • Lead Ads（即時表單）：點擊後在 Meta 內直接彈出表單，不離開 App 即可留資。
  • Click-to-Message：點擊後直接打開 WhatsApp 或 Messenger 對話框，立即開始問答。

第二層：真實畫面預覽（示例）

Feed：出現在 FB/IG 貼文流，點擊後進入落地頁、表單或訊息入口。
Stories：出現在 FB/IG 限時動態全屏，點擊後直接跳轉 CTA 目的地。
Reels：出現在 FB/IG Reels 短片流，點擊後導向訊息或落地頁。
Click-to-Message：看到廣告後點「Send Message」，直接進入 WhatsApp/Messenger 對話。
Lead Ads：點 CTA 後在 Meta 內彈出即時表單，提交後進入名單池。

第三層：本案主選與備選的用戶路徑

主選路徑：Click-to-Message（WhatsApp/Messenger）為主
看到廣告 -> 點 CTA -> 打開 WhatsApp / Messenger 對話 -> 以 2-3 個篩選問題確認是否有效諮詢 -> 進入跟進記錄或 CRM

用途：先用對話拿到高意向樣本，再按有效諮詢率決定放量。

預設篩選問題（3 問）：
Q1 你的店型是茶餐廳/快餐/外賣店/其他？
Q2 你現在用緊邊套 POS 或點餐系統？（沒有/有，品牌可選填）
Q3 你最想先解決哪一件事？（外賣平台訂單整合/掃碼點餐/收銀出單/成本上限）

有效諮詢判定（預設）：店型符合目標，且表達明確需求或願意了解試用/演示即判定有效；不符合則判定無效，並在回流字段記錄無效原因（枚舉）。

輔選路徑：Lead Ads（補量與穩定）
看到廣告 -> 點 CTA -> 彈出 Meta 即時表單 -> 提交 -> 進入名單池或 CRM -> 後續跟進

切換策略：當主選 CPL 過高或有效諮詢率偏低時，用 Lead Ads 補足樣本量並穩定成本。

五、風險與歸因可視化

P1/P2 風險優先級

  • P1：諮詢量不足（直接影響闸門推進與樣本有效性）
  • P2：有效諮詢占比偏低（名單多但可成交線索不足）

未達標歸因流程

  • 步驟 1：流量（諮詢量） -> 是否先天量不足
  • 步驟 2：質量（有效諮詢率） -> 是否流量結構失衡
  • 步驟 3：後鏈路轉化（試用/激活） -> 以「進入試用或註冊登入」「綁定打印機與支付終端」「完成菜單錄入」判斷是否卡住

三、Meta 廣告形式管理層速覽（v1.6.2）

第一層：一句話畫面（看到什麼，點了去哪）

Reels 與 Stories 在 Facebook 與 Instagram 都有對應版位，投放時可分別覆蓋 Facebook Reels/Stories 與 Instagram Reels/Stories（視設定而定）。

  • Feed：像一般滑動貼文中的贊助內容，點擊後通常進入表單或訊息頁。
  • Stories：全螢幕直式畫面，點擊後直接跳轉到表單或對話入口。
  • Reels：短影片流中的插入廣告，點擊後多數導向訊息或落地頁。
  • Lead Ads（即時表單）：點擊後在 Meta 內直接彈出表單，不離開 App 即可留資。
  • Click-to-Message：點擊後直接打開 WhatsApp 或 Messenger 對話框，立即開始問答。

第三層：本案主選與備選的用戶路徑

主選路徑：Click-to-Message（WhatsApp/Messenger）為主
看到廣告 -> 點 CTA -> 打開 WhatsApp / Messenger 對話 -> 以 2-3 個篩選問題確認是否有效諮詢 -> 進入跟進記錄或 CRM

用途：先用對話拿到高意向樣本，再按有效諮詢率決定放量。

輔選路徑：Lead Ads（補量與穩定）
看到廣告 -> 點 CTA -> 彈出 Meta 即時表單 -> 提交 -> 進入名單池或 CRM -> 後續跟進

切換策略：當主選 CPL 過高或有效諮詢率偏低時，用 Lead Ads 補足樣本量並穩定成本。

五、風險與歸因可視化

P1/P2 風險優先級

  • P1：諮詢量不足（直接影響闸門推進與樣本有效性）
  • P2：有效諮詢占比偏低（名單多但可成交線索不足）

未達標歸因流程

  • 步驟 1：流量（諮詢量） -> 是否先天量不足
  • 步驟 2：質量（有效諮詢率） -> 是否流量結構失衡
  • 步驟 3：轉化（試用/激活） -> 是否後鏈路承接問題
